int32 ChameleonInit (  OSS_HANDLE *  osh,
int  pciBus,
int  pciDev,
CHAMELEON_HANDLE **  chahP
) 
 

routine to init Chameleon device

Before calling this routine, the BARs of the PCI device must be filled!

It creates the handle for that device.

This routine

  • Checks the presence of the PCI device (not the PCI device/vendor ID!)
  • Reads the base address registers
  • Checks if the configuration table contains the proper sync word
  • Determines the number of units in the configuration table I
    Parameters:
    osh OSS handle, returned by OSS_Init
    pciBus PCI Bus Id (contains pciDomain in upper 8 bit!)
    pciDev PCI Device Id
    chahP pointer to variable where handle will be stored
    Returns:
    0=ok, or error number CHAMELEON_xxx

    *chahP = handle

int32 ChameleonUnitFind (  CHAMELEON_HANDLE *  h,
int  modCode,
int  instance,
CHAMELEON_UNIT *  info

Search configuration table for unit.

Searches the configuration table for the specified module code and instance number. For example, to search Frodo UART, 2nd instance, pass modCode as 7 and instance as 1. If modCode and/or instance is specified as -1, the function ignores that parameter

Compatibility
 If used with Chameleon-V2 tables, not all instances may be found because Chameleon-V2 tables may use one and the same instance number several times within the tables of one chameleon system (because pluggable busses).

Parameters:
h CHAMELEON_HANDLE returned from ChameleonInit()
modCode module code to search (CHAMELEON_xxx)
instance nth instance to search for (0..n)
info ptr to struct where information will be stored
Returns:
0 or error code
