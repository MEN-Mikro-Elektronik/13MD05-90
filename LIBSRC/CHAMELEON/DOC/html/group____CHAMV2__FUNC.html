_STATIC_ int32 BridgeIdent (  CHAMELEONV2_HANDLE *  chah,
u_int32  idx,
CHAMELEONV2_BRIDGE *  bridge
) 
 

Chameleon-V2 routine to identify all bridges.

Retrieve information about all bridges implemented in the chameleon device (see CHAMELEONV2_BRIDGE).

The idx parameter is used as an consecutive index and specifies for which bridge the information should be returned in bridge. The function returns CHAMELEONV2_NO_MORE_ENTRIES if no (more) bridge with the given index exist.

Compatibility
 Supports Chameleon-V0/V1 and Chameleon-V2 devices. The function returns CHAMELEONV2_NO_MORE_ENTRIES for Chameleon-V0/V1 devices.

Parameters:
chah chameleon handle returned from InitXx()
idx index of bridge to query (0..n)
bridge ptr to CHAMELEONV2_BRIDGE struct where information will be stored
Returns:
0=ok, or chameleon error code

_STATIC_ int32 InitInside (  OSS_HANDLE *  osh,
void *  tblAddr,
CHAMELEONV2_HANDLE **  chahP
) 
 

Chameleon-V2 init routine for chameleon devices at non PCIbus location.

This is the initialization function for callers inside a chameleon device or when the chameleon device resides at a non PCIbus (e.g. LPC bus). The chameleon device must be addressed by the tblAddr parameter.

The function returns a handle for the specified chameleon device.

This routine

  • Checks the presence of the specified chameleon device
  • Checks if the configuration table contains the sync word
  • Determines the number of units in the configuration table

Compatibility
 Supports Chameleon-V2 devices only. The function returns CHAMELEONV2_TABLE_REV_UNSUP for Chameleon-V0/V1 devices.

Parameters:
osh OSS handle, returned by OSS_Init
tblAddr (unmapped) address of the chameleon table for the bus where the caller resides
chahP pointer to variable where chameleon handle will be stored
Returns:
0=ok, or chameleon error code

_STATIC_ int32 InitPci (  OSS_HANDLE *  osh,
u_int32  pciBus,
u_int32  pciDev,
u_int32  pciFunc,
CHAMELEONV2_HANDLE **  chahP
) 
 

Chameleon-V2 init routine to access a chameleon device from PCI side.

This is the initialization function for callers outside a chameleon device that resides on a PCI bus. The chameleon device must be addressed by the pciBus, pciDev, pciFunc parameters.

The function returns a handle for the specified chameleon device.

This routine

  • Checks the presence of the specified chameleon device
  • Reads the base address registers
  • Checks if the configuration table contains the sync word
  • Determines the number of units in the configuration table
  • Maps the address space of the config table (if required for the OS)

Compatibility
 Supports Chameleon-V0/V1 and Chameleon-V2 devices.

Parameters:
osh OSS handle, returned by OSS_Init
pciBus PCI Bus Id (0..255, upper 8 bit contain pci domain!)
pciDev PCI Device Id (0..31)
pciFunc PCI Function Id (0..7)
chahP pointer to variable where chameleon handle will be stored
Returns:
0=ok, or chameleon error code

int32  idx,
CHAMELEONV2_FIND  find,
CHAMELEONV2_UNIT *  unit,
CHAMELEONV2_BRIDGE *  bridge,
CHAMELEONV2_CPU *  cpu
) 
 

Chameleon-V2 routine to search for a General Device unit.

Searches the chameleon device for a unit/bridge/cpu with the specified characteristic(s). The demanded characterstics must be specified with the CHAMELEONV2_FIND structure. The following table shows which CHAMELEONV2_FIND characteristic applies to unit/bridge/cpu:

  	characteristic  |  unit  | bridge |   cpu
  	----------------+--------+--------+--------
  	devId           |    x   |    x   |    x
  	variant         |    x   |    x   |    x
  	instance        |    x   |    x   |    x
  	busId           |    x   |    x   |    x
  	group           |    x   |    x   |    x
  	bootAddr        |    -   |    -   |    x

A charactersitic can be ignored by specifying -1. If none characteristic is specified, all implemented units/bridges/cpus will be found.

With the 'flags' member of the CHAMELEONV2_FIND structure, the search behaviour can be adjusted:
The CHAMELEONV2_FF_BRGALL flag causes that, besides all matching modules, all units and bridges within a bridged destination window of a matching bridge will be found, regardless if the units and bridges within the window match or match not with the specified find characteristics. Note: Only units and bridges at the next destination bus of the bridge will be considered. Units and bridges at further busses behind further bridges within a matching bridge window are not considered.

The function searches after a unit/bridge/cpu if a ptr to the belonging CHAMELEONV2_UNIT/BRIDGE/CPU structure is provided. If NULL is specified for unit, bridge or cpu this instance types will not be querried.

Example: InstanceFind( chah, idx, find, unit, NULL, cpu ) searches for units and cpus but not for bridges that match the specified characteristics.

The idx parameter is used as an consecutive index and specifies for which found unit/bridge/cpu match the information should be returned.

The function indicates with the return value which instance (unit/bridge/cpu) was actually found or if no (more) instances with the specified characteristic(s) and index was found.:

  • CHAMELEONV2_UNIT_FOUND : unit points to the found unit information
  • CHAMELEONV2_BRIDGE_FOUND : bridge points to the found bridge information
  • CHAMELEONV2_CPU_FOUND : cpu points to the found cpu information
  • CHAMELEONV2_NO_MORE_ENTRIES : no unit/bridge/cpu found

Compatibility
 Supports Chameleon-V0/V1 and Chameleon-V2 devices. For Chameleon-V0/V1 devices:

  • searches only for units
  • the variant, busId and group parameters are alway 0
  • see CHAMELEONV2_UNIT struct restrictions

Parameters:
chah chameleon handle returned from InitXx()
idx index of unit to query (0..n)
find CHAMELEONV2_FIND strucure with the charateristics to search
unit ptr to CHAMELEONV2_UNIT struct or NULL
bridge ptr to CHAMELEONV2_BRIDGE struct or NULL
cpu ptr to CHAMELEONV2_CPU struct or NULL
Returns:
0=ok, or chameleon error code

Chameleon-V2 routine to identify all Chameleon tables.

Retrieve information about all Chameleon tables within the Chameleon table tree (see CHAMELEONV2_TABLE).

The idx parameter is used as an consecutive index and specifies for which table the information should be returned in table. The function returns CHAMELEONV2_NO_MORE_ENTRIES if no (more) unit with the given index exist.

Note: Use idx=0 to get the information about the table of the bus where the caller resides.

Compatibility
 Supports Chameleon-V0/V1 and Chameleon-V2 devices. See CHAMELEONV2_TABLE struct restrictions for Chameleon-V0/V1 devices.

Parameters:
chah chameleon handle returned from InitXx()
idx index of table to query (0..n)
table ptr to CHAMELEONV2_TABLE struct where information will be stored
Returns:
0=ok, or chameleon error code
