Chameleon Library Documentation

2.80

The Chameleon library provides functions to get information about the units, bus architecture and structure of Chameleon devices implemented within FPGAs.

This common library is used in MENMON, BSPs and drivers to perform common actions on Chameleon devices and is used as an information data base.

Chameleon Library Glossary:
A Chameleon FPGA can implement one ore more Chameleon devices.

A Chameleon Device is the whole chunk of all Chameleon sub-devices (e.g. units, bridges, CPUs) connected via internal buses.

A Chameleon Function is one complex function built with several Chameleon units (display controller, bit blitter, SDRAM window).

A Chameleon Unit is one simple function (e.g. an SDRAM window) within a Chameleon device.

A Chameleon Table Tree consists of all Chameleon cables linked together which describes a whole Chameleon device (all internal buses of the Chameleon device).

A Chameleon Table describes all Chameleon sub-devices (e.g. units, bridges, CPUs) on one Chameleon device internal bus.

Chameleon Library Functions:

The library supports Chameleon-V0/V1 and Chameleon-V2 tables with the following function groups:

  • The Chameleon-V0 Functions were implemented for the first Chameleon table version (V0) and updated for Chameleon-V1 tables but supports also Chameleon-V2 tables (by using internally the Chameleon-V2 functions).

  • The Chameleon-V2 Functions were added to provide the extended information from Chameleon-V2 tables but supports also Chameleon-V0/V1 tables (by using internally the Chameleon-V0 functions).

  • The Chameleon Code Functions can be used to convert between Chameleon-V0/V1 module codes and Chameleon-V2 device IDs and to get the name of a unit for a module code or device ID.

  • ChameleonLoadxxx() Functions
    The board specific ChameleonLoadxxx() (re-)loads the FPGA with the specified code. It is a special function in that it doesn't require a handle (it can be called without calling V0::ChameleonInit(), V2::InitPci(), V2::InitInside() before). fpgaCode must point to the FPGA code's header followed by the FPGA code.

Unit Test

For library verification (Chameleon-V0 and Chameleon-V2 functions) in User Mode without real hardware, the CHA_UT unit test is provided. See cha_ut.c

The CHA_UT unit test must always be executed with success after each modification of the library!
