int32 UOS_SigInstall (  u_int32  sigCode  ) 
 

Install a signal to be received.

This function assigns the signal sigCode to the users signal handler. If it is not called, it isn't garanteed that the signal will be handled by the users signal handler

If specified signal is already installed, the function returns an ERR_UOS_BUSY error.

If specified signal is not allowed, the function returns an ERR_UOS_ILL_SIG error.

If signal handling not initialized, the function returns an ERR_UOS_NOT_INIZED error.

See Using USR_OSS Signals for a general description of USR_OSS signal handling.

Returns:
0 on success or ERR_UOS_xxx error code on error. On failure, UOS global errno is set to the error code

Linux notes:
See Using USR_OSS Signals on Linux to see how signals handling is implemented for Linux.
Linux notes:
No ERR_UOS_BUSY error is returned if signal already installed.

int32 UOS_SigMask (  void   ) 
 

Mask all signals.

Blocks all signals (at least those installed by UOS_SigInstall()). All incomming signals are queued.

Remarks:
UOS_SigMask and UOS_SigMask() do not need to support nesting. I.e. if you call UOS_SigUnMask() signals will be unmasked regardless how often you called UOS_SigMask() before.
Returns:
0 on success or ERR_UOS_xxx error code on error. On failure, UOS global errno is set to the error code

int32 UOS_SigWait (  u_int32  msec,
u_int32 *  sigCodeP
) 
 

Wait until signal received.

This function waits until a signal is received or a timeout occurs. If a signal has been received, its value is store in sigCodeP and in addition, the users signal handler is called (if installed).

The function checks if any signal is already queued:

  • Signal is already queued: The function reads the signal from the queue and calls the user signal handler (if installed).
  • Queue is empty: The function unmasks all signals and puts the process into sleep state until a new signal received.

If no signal was received within the timeout period, the function returns with error ERR_UOS_TIMEOUT.

If signal handling not initialized, the function returns an ERR_UOS_NOT_INIZED error.

On some OSes, if the process received a deadly signal (i.e. was killed), the function returns with error ERR_UOS_ABORTED and a valid signal code in sigCodeP.

Notes:

  • Signals must be masked before calling this function.
  • Signals will be masked when this function returns.
  • Given timeout will be rounded up to system ticks

Parameters:
msec wait timeout (0=endless) [msec]
sigCodeP pointer to var where received signal will be stored
Returns:
0 on success or ERR_UOS_xxx error code on error. On failure, UOS global errno is set to the error code

Linux notes:
ERR_UOS_ABORTED will never be returned. Any signal not installed by UOS_SigInit will cause the standard signal handler to be invoked.
