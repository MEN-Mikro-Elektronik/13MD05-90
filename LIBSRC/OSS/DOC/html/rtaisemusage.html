RTAI notes to OSS semaphores

This implementation uses native RTAI semaphores.

This works fine. However when the semaphore's value reaches 0xffff we run into troubles because this is the same value as SEM_ERR. Hopefully, the RTAI guys will fix this some day.

Semaphore waits cannot be aborted by signals under RTAI.
