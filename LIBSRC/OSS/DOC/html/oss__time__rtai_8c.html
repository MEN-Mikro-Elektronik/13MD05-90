int32 OSS_RtaiDelay (  OSS_HANDLE *  oss,
int32  msec
) 
 

Let process sleep for specified time.

Suspends the calling process/task for at least the specified time. Time may be rounded up according to ticker resolution.

A msec value of <= 0 shall delay the process for the smallest possible time.

Remarks:
All signals during sleep will be ignored. It does not return before the specified time has elapsed.

Don't use in IRQ or Alarm routines.

Parameters:
oss IN OSS handle, returned by OSS_Init()
msec IN number of milliseconds to delay
Returns:
actual time slept in milliseconds

RTAI notes:
The behaviour of this function depends on the rtMode flag passed to the OSS_Init() function. Either the standard Linux kernel or RTAI implementation is called. For Linux implementation, see OSS_Delay().
RTAI notes:
You must have called OSS_RtaiTimingInit() before!
See also RTAI timer notes.

OSS_RtaiMikroDelay

u_int32  usec
) 
 

Delay execution of a process by using a busy-loop.

Delays the calling process for at least usec by wasting CPU time in a busy loop. It can be called from interrupt or alarm context.

Remarks:
usec shall be in range 0..1000 microseconds.

OSS_MikroDelayInit() must be called before this routine can be used.

Parameters:
oss IN OSS handle, returned by OSS_Init()
microSec IN number of microseconds to delay

RTAI notes:
The behaviour of this function depends on the rtMode flag passed to the OSS_Init() function. Either the standard Linux kernel or RTAI implementation is called. For Linux implementation, see OSS_MikroDelay().
RTAI notes:
rt_busy_sleep() is used. See also RTAI timer notes.

u_int32 OSS_RtaiTickGet (  OSS_HANDLE *  oss  ) 
 

Get the current system tick.

The OSS tick is a 32 bit value that is incremented at a rate determined by OSS_TickRateGet(). The counter always uses the full 32 bits and wraps from 0xffffffff to 0x00000000.

Remarks:
If interrupts are masked, the tick may no longer be incremented.
Parameters:
oss IN OSS handle, returned by OSS_Init()
Returns:
current system tick value.

RTAI notes:
The behaviour of this function depends on the rtMode flag passed to the OSS_Init() function. Either the standard Linux kernel or RTAI implementation is called. For Linux implementation, see OSS_TickGet().
Linux notes:
uses rt_get_time().
RTAI notes:
You must have called OSS_RtaiTimingInit() before!
See also:
OSS_RtaiTickRateGet See also RTAI timer notes.
In both RTAI classic and Xenomai behaviour is mode dependent. In periodic mode, clock ticks are expressed as periodic jiffies. In oneshot mode, clock ticks are expressed as nanoseconds.

int32 OSS_RtaiTickRateGet (  OSS_HANDLE *  oss  ) 
 

Get the tick rate.

Get the rate at which the OSS tick is incremented in ticks per seconds.

Remarks:
this function shall return a value between 50 and 1000000; i.e. 20ms to 1us.
Parameters:
oss IN OSS handle, returned by OSS_Init()
Returns:
tick rate in Hz

RTAI notes:
The behaviour of this function depends on the rtMode flag passed to the OSS_Init() function. Either the standard Linux kernel or RTAI implementation is called. For Linux implementation, see OSS_TickRateGet().
RTAI notes:
To avoid too huge tickrates in oneshot mode, the RTAI tickrate is devided by 4096. For example, in oneshot mode on a 1 GHz pentium, the tickrate value would be 244140 (1E9/4096). For periodic mode, the tickrate corresponds to the frequency of one RTAI scheduler period.
RTAI notes:
You must have called OSS_RtaiTimingInit() before!
See also:
OSS_RtaiTickGet See also RTAI timer notes.

void OSS_RtaiTimingInit (  int  oneShotMode  ) 
 

Initialize OSS RTAI timing.

RTAI specific call. Must be called after start_rt_timer() and every time the RTAI timer mode is changed.

Required since OSS must know wether RTAI is in oneshot or periodic mode and there is currently no RTAI API function to query this mode. For use with Xenomai, rt_timer_inquire is used.

Sets up globals OSS_tmrOneShotMode and OSS_tmrPeriod

See also RTAI timer notes.

Parameters:
oneShotMode IN 0=periodic mode, 1=oneshot mode
