RTAI notes to OSS alarms

Under RTAI, alarms are implemented using a separate task for each alarm routine.

User's alaram function is called in the context of this task which runs at a fixed priority of 1 (change TASK_PRIORITY define if you don't like this).

Cyclic alarms are implemented using the rt_task_make_periodic()a feature.

Under Xenomai, Alarms are part of the native API and therefore MDIS Alarm Services are implemented using these functions:

rt_alarm_create Create an alarm object from kernel space. rt_alarm_delete Delete an alarm. rt_alarm_start Start an alarm. rt_alarm_stop Stop an alarm.

The granularity of alarm cycles depends on the scheduling algorithm of RTAI.
