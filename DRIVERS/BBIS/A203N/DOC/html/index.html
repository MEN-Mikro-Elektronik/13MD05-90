A203N BBIS Driver Documentation

This is the documentation of the BBIS driver for the A203N board.

The A203N is an M-Module carrier board for universal I/O on the VMEbus. The A203N is a VME64 master and slave card and supports four D16/D32 M-Modules. Additional, the A203N provides onboard trigger capabilities and an onboard temperature sensor.

Please refer to the A203N hardware documentation for further details about the A203N hardware.

Slot assignment:

Slot assignment:

     slot   |  device
    --------+--------------------------------
          0 |  M-Module slot 0
          1 |  M-Module slot 1
          2 |  M-Module slot 2
          3 |  M-Module slot 3
     0x1000 |  Trigger device for TRIGA/TRIGB lines
     0x1001 |  Temperature sensor (LM75 on I2C interface)

Notes:

  • Note: The DMA features of the A203N are not supported!


Variants

In order to support different CPU boards and to achieve maximum performance, the A203N driver can be built in some variants at compilation time:

    Driver              Variant Description
    --------            --------------------------------
    Standard            Non-swapped register access,
    _sw                 Swapped register access

Here are some combinations of MEN CPU and carrier boards together with the required variants:

    CPU                     Driver Variant
    ----------------        --------------
    MEN D2 (Pentium CPCI)   Standard
    MEN F1 (PPC CPCI)       _sw


Interrupts

The driver supports INTA interrupts from the M-Module. The M-Module’s interrupt can be enabled/disabled through the M_MK_IRQ_ENABLE SetStat code.


A203N Descriptor Entries

The bbis initialization routine A203N_Init() decodes the following entries ("keys") in addition to the general descriptor keys:

Descriptor entry Description Values
VME16_ADDR_MMOD08 A16 base addr on VMEbus for A08 M-Mod space 0x0XXX..0xfXXX
VME24_ADDR_MMOD08 A24 base addr on VMEbus for A08 M-Mod space 0x000XXX..0xfffXXX
VME32_ADDR_MMOD24 A32 base addr on VMEbus for A24 M-Mod space 0x0XXXXXXX..0xfXXXXXXX
IRQ_VECTOR Irq vectors for module 0..3 <vec0>,<vec1>,<vec2>,<vec3>
where <vecX>=0x00..0xff
IRQ_LEVEL Irq levels for module 0..3 <lev0>,<lev1>,<lev2>,<lev3>
where <levX>=1..7
IRQ_PRIORITY Irq priorities for module 0..3 <pri0>,<pri1>,<pri2>,<pri3>
where <priX>=0x00..0xff
Default: 0,0,0,0
PNP_FORCE_FOUND "Force found" for module slot 0..3:
0: only a recognized M-Module will be reported
1: an unknown M-Module will be reported regardless if there is one plugged 		<fofo0>,<fofo1>,<fofo2>,<fofo3>
where <fofoX>=0 or 1
Default: 0,0,0,0

A08 M-Module support (always)
VME16_ADDR_MMOD08 or VME24_ADDR_MMOD08 must be specified to get access to the board's control register and to support the A08 M-Module address space.
Notes:

  • VME16_ADDR_MMOD08 and VME24_ADDR_MMOD08 cannot be specified together.
  • If VME16_ADDR_MMOD08 is specified, the "short non-privileged access mode (0x29)" will be used on the VMEbus.
  • If VME24_ADDR_MMOD08 is specified, the "Standard non-privileged data access mode (0x39)" will be used on the VMEbus.

A24 M-Module support (optional)
The bbis supports access to the A24 M-Module address space if VME32_ADDR_MMOD24 is specified.
Notes:

  • The "Extended non-privileged data access mode (0x09)" will be used on the VMEbus.
