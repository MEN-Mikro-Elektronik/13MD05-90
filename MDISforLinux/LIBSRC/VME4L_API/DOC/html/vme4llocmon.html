Location monitor support

VME4L supports VME location monitor(s).

It provides direct access to the location monitor registers using VME4L_LocMonRegRead() and VME4L_LocMonRegWrite().

This allows maximum flexibility, although you must know the register layout of the location monitor.

To get informed about a location monitor hit, install a signal similar to mailbox signals:

      // install signal for second location monitor
      VME4L_SigInstall( G_fd, VME4L_IRQVEC_LOCMON(1),
                              VME4L_IRQLEV_LOCMON(1), 38, VME4L_IRQ_ENBL );

Notes for VME-bridge on A13/A15:
Supports two location monitors.
The register numbers to pass to VME4L_LocMonRegRead() and VME4L_LocMonRegWrite() are:

  reg    PLD register
  ----   -------------
    0	 Location monitor 0 control
    1	 Location monitor 1 control
    2	 Location monitor 0 address
    3	 Location monitor 1 address
When reg 0 or 1 is written, the interrupt enable bit supplied by user is ignored.

See HW manual for more details

Notes for VME-bridge on A12/B11:
Locations monitors are not supported
Notes for VME-bridge on A17/A19/A20:
Supports four locations (each location is 8 bytes). Address bits 3 and 4 are used to define the specific location.
The register numbers to pass to VME4L_LocMonRegRead() and VME4L_LocMonRegWrite() are:

  reg    TSI148 register
  ----   -------------
    0	 LMBAU (Location Monitor Base Address Upper)
    1	 LMBAL (Location Monitor Base Address Lower)
    2	 LMAT (Location Monitor Attribute Register)

See chapter 2.3.6.1 Location Monitor in TSI148 User Manual for more details
