int32 OSS_RtaiSigCreate (  OSS_HANDLE *  oss,
int32  signal,
OSS_SIG_HANDLE **  sigP
) 
 

Create signal handle.

This creates an OS specific handle to store an identifier for the calling task. To send the signal, use OSS_SigSend().

This routine must be called in the context of the process/task that wishes to receive the signal.

Remarks:
Don't use in IRQ or Alarm routines.
Parameters:
oss IN OSS handle, returned by OSS_Init()
signal IN signal code to install
sigP OUT will receive created signal handle. Must be passed to further OSS_SigXXX() routines.
Returns:
0 on success or ERR_OSS_xxx error code on error

See RTAI notes to OSS signals for more info.

RTAI notes:
The behaviour of this function depends on the rtMode flag passed to the OSS_Init() function. Either the standard Linux kernel or RTAI implementation is called. For RTAI implementation, see OSS_RtaiSigCreate().
See also:
OSS_RtaiSigRemove, OSS_RtaiSigSend

int32 OSS_RtaiSigRemove (  OSS_HANDLE *  oss,
OSS_SIG_HANDLE **  sigP
) 
 

Destroy semaphore handle.

Frees all resources allocated by OSS_SigCreate(). The function fails if the calling process id is not equal to the installing process.

Remarks:
Don't use in IRQ or Alarm routines.
Parameters:
oss IN OSS handle, returned by OSS_Init()
sigP IN pointer to signal handle to remove OUT *sigP set to NULL
Returns:
0 on success or ERR_OSS_xxx error code on error

See RTAI notes to OSS signals for more info.

RTAI notes:
The behaviour of this function depends on the rtMode flag passed to the OSS_Init() function. Either the standard Linux kernel or RTAI implementation is called. For RTAI implementation, see OSS_RtaiSigRemove().
See also:
OSS_RtaiSigCreate, OSS_RtaiSigSend

int32 OSS_RtaiSigTaskRegister (  MBX *  mbx  ) 
 

Register task for signal handling.

called from UOS_SigInit() to register a task for signal handling. Creates an OSS_RTAI_SIG_TASK structure and adds it to the G_sigTaskTbl list. In Xenomai the Signals are passed to the Tasks via message queues.

Parameters:
mbx IN mailbox to use for that task
Returns:
0 on success or ERR_OSS_xxx error code on error:
  • ERR_OSS_BUSY_RESOURCE: task already registered
