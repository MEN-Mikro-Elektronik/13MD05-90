Wait for semaphore.

See Linux notes to OSS semaphores for more info.

Linux notes:
  • all signals < SIGRTMIN are assumed to be deadly.
  • LL drivers trying to ignore all signals while waiting for a signal will cause a busy loop under linux, because the scheduler awakes a process always immediately when a non masked signal is pending.
