int32 OSS_Delay (  OSS_HANDLE *  oss,
int32  msec
) 
 

Let process sleep for specified time.

Linux notes:
The behaviour of this function depends on the rtMode flag passed to the OSS_Init() function. Either the standard Linux kernel or RTAI implementation is called. For RTAI implementation, see OSS_RtaiDelay().
Linux notes:
If a signal occurs during sleep, all signals are masked until sleep has finished.
See also:
OSS_MikroDelay

Linux notes:
The timer used has a resolution of aprox 10msec. This means that the delay will be at least between 10 and 20 msec
See also:
OSS_MikroDelay

int32 OSS_MikroDelayInit (  OSS_HANDLE *  oss  ) 
 

Initialisation routine for OSS_MikroDelay().

Linux notes:
For Linux and RTAI, this is a no-op routine.
See also:
OSS_MikroDelay

Linux notes:
Since we don't have a timer that we can use, we try to measure a loop counter three times. The minimum time is used and then doubled to get at least the required delay.
