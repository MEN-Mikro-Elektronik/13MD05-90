OSS module for Linux/RTAI Documentation

2.15

This is the documentation of the MEN OSS module (Operating System Services) for Linux and/or RTAI kernel mode.

Refer to the OSS Common Specification for the plain common specification.

Under Linux, OSS is implemented as a loadable kernel module, named men_oss or men_oss_dbg.

Module Parameters:

  • #oss_pci_slot_devnbrs

RTAI support

If the OSS module is compiled with switch MDIS_SUPPORT_RTAI, OSS can be instanciated either for use with standard Linux kernel drivers or RTAI kernel mode applications, see OSS_Init(). Therefore, the OSS module exists only once; switching between Linux/RTAI functions is done dynamically for each instance of OSS.

The following functions switch dynamically between Linux and RTAI implementations:

  • Memory allocation: OSS_MemGet(), OSS_MemFree()
  • Interrupt related: OSS_IrqMask(), OSS_IrqUnMask()
  • Time related: OSS_Delay(), OSS_MikroDelay(), OSS_TickRateGet(), OSS_TickGet(), OSS_MikroDelayInit()
  • Semaphores: OSS_SemCreate(), OSS_SemRemove(), OSS_SemWait(), OSS_SemSignal()
  • Signals: OSS_SigCreate(), OSS_SigRemove(), OSS_SigSend(), OSS_SigInfo()
  • Alarms: OSS_AlarmCreate(), OSS_AlarmRemove(), OSS_AlarmSet(), OSS_AlarmClear(), OSS_AlarmMask(), OSS_AlarmRestore()

For the rest of the functions, the standard Linux implementations are called, even when OSS is instanciated for RTAI mode.

Special OSS calls under Linux

  • Initialisation and termination: OSS_Init(), OSS_Exit()
  • RTAI timing mode initialisation: OSS_RtaiTimingInit()
  • interface routines to USR_OSS (should not be called by user): OSS_RtaiMsToCount(), OSS_TaskToSigEntry(), OSS_RtaiSigTaskRegister(), OSS_RtaiSigTaskUnregister()
